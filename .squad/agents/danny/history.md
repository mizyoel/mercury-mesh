### 2026-03-31: Org Hierarchy Architecture Review

- **Key decision:** CEO IS the coordinator in hierarchical mode, not a separate agent. A separate CEO creates a spawn-chicken-egg problem (nothing can spawn the coordinator) and doubles routing latency. The coordinator already routes, delegates, and decides — it just needs new routing logic behind a feature flag.
- **Key decision:** Department leads are routing metadata, not mandatory agent spawns. Eliminates 3-hop latency (CEO→lead→member). Leads only spawn for escalation or cross-department coordination.
- **Pattern:** Feature flags on coordinator behavior (orgMode in config.json) are the safest way to introduce major behavioral changes — rollback is one boolean flip.
- **Pattern:** Additive schemas with optional fields preserve backward compatibility. Always design for "missing field = default/flat behavior."
- **Gotcha:** squad.agent.md is the hardest file to change and must be treated as a primary artifact in any plan that changes coordinator behavior. Plans that list it as "affected" without specifying exact changes are incomplete.
- **Gotcha:** Workflow files (squad-triage.yml, sync-squad-labels.yml, squad-issue-assign.yml, squad-heartbeat.yml) parse `## Members` by column position. Adding columns is safe (ignored by position-based parsers) but renaming/reordering columns breaks them.
- **File paths:** `.squad/org/structure.json` (hierarchy definition), `.squad/config.json` (orgMode flag), `.squad/org/migration.md` (migration checklist), `.squad/org/dashboard.md` (generated by Ralph/Scribe)
- **Label strategy:** During migration, `squad:{member}` labels remain workflow triggers. `dept:*` labels are routing metadata only. No precedence conflict because they serve different purposes.
- **Decision scoping:** Single `decisions.md` with scope tags (`org`, `dept:{name}`) rather than per-department decision files. Preserves visibility, avoids fragmentation.
